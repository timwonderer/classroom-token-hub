# CTH Build Specification
## Feature: Teacher Decisions Log (Student View)

---

# Purpose

Provide students with a **read-only record of teacher-initiated state mutations affecting them**.

The page allows students to understand **when and how a teacher changed rules or records that affect their account**.

This log **does not include system automation or financial transactions**.

Financial changes remain in **Transaction History**.

---

# Design Principles

## 1. Governance Events Only

The log records **teacher decisions**, not system activity.

Examples of valid events:

- Pay rate updated
- Ticket resolved
- Ticket escalated
- Ticket reopened
- Attendance record deleted
- Insurance policy updated
- Insurance coverage manually cancelled

---

## 2. Financial Events Excluded

Anything generated automatically by the system must **not appear** in this log.

Examples excluded:

- Payroll deposits
- Store purchases
- Overdraft fees
- Rent payments
- Tax deductions
- Insurance auto-cancellation

These remain visible only in **Transaction History**.

---

## 3. Read Only

Students cannot edit or interact with these events.

The page is purely informational.

---

## 4. Class Scoped

Events are visible only within the currently selected class.

If a student loses class membership, the log disappears with the class.

# Data Source Strategy

No new database table will be created.

The page is generated by **querying existing mutation logs**.

Potential sources include:

- ticket_history
- attendance_adjustments
- teacher_balance_adjustments
- policy_updates

Each source query must normalize results into a common structure.

---

# Normalized Query Format

Each event returned to the UI must provide:

- timestamp
- action_type
- summary
- details
- source_table
- source_id

This structure exists only in the **query result**, not as a stored table.

---

# Example Query Pattern

## Job Assignment History

SELECT
    updated_at AS timestamp,
    'JOB_ASSIGNED' AS action_type,
    'Job assigned' AS summary,
    job_id AS source_id,
    'jobs' AS source_table
FROM job_history
WHERE student_id = ?
AND class_id = ?

---

## Ticket Resolution

SELECT
    resolved_at AS timestamp,
    'TICKET_RESOLVED' AS action_type,
    'Ticket resolved' AS summary,
    ticket_id AS source_id,
    'tickets' AS source_table
FROM tickets
WHERE student_id = ?
AND class_id = ?
AND resolved_by_teacher = TRUE

---

## Attendance Correction

SELECT
    updated_at AS timestamp,
    'ATTENDANCE_RECORD_DELETED' AS action_type,
    'Attendance record removed' AS summary,
    attendance_id AS source_id,
    'attendance' AS source_table
FROM attendance_changes
WHERE student_id = ?
AND class_id = ?
AND modified_by_teacher = TRUE

---

# Final Event Aggregation

The event page aggregates normalized events using UNION:

SELECT *
FROM (
    job_history_events
    UNION ALL
    ticket_events
    UNION ALL
    attendance_events
    UNION ALL
    admin_adjustments
) teacher_decisions
ORDER BY timestamp DESC
LIMIT 100

---

# Security Rules

Students may only view events where:

student_id = current_user_student_id  
class_id = active_class_id

Students cannot access events belonging to other students.

---

# Performance

Indexes recommended on source tables:

(student_id, class_id, updated_at)

Teacher decisions are expected to be **low-frequency events**, so performance impact should be minimal.

---

# MVP Acceptance Criteria

Feature is complete when:

- The feature exist as tab in help and support
- Events appear in chronological order
- Only teacher-initiated state mutations appear
- Financial/system events do not appear
- Clicking a row shows detailed information
- Students cannot edit or delete entries
