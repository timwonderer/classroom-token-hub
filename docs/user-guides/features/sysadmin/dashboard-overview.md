# Dashboard and Logs

## Overview
As a System Administrator, your primary tool for monitoring platform health is the global Dashboard. The main dashboard gives you a high-level overview of important metrics across the platform. Use this daily to check for spikes in user activity or errors.

## Step-by-Step Instructions

### Examining System Logs
1. Navigate to **System Logs** in your Sysadmin sidebar.
2. View general activity generated by teachers and students to monitor usage patterns.

### Examining Error Logs
1. Navigate to **Error Logs**.
2. Drill down into caught errors to identify broken features or recurring issues.
3. For a unified timeline, check the **Combined Logs** page.

### Using the Grafana Integration
The system includes an integrated Grafana dashboard for advanced visualizations:
1. Select **Grafana Dashboard** from the navigation.
2. Ensure you are authenticated through the system proxy.
3. Review specific analytics related to database performance and traffic.

## Important Notes
> [!TIP]
> **Stay proactive:** Check logs daily to catch potential issues before users report them.

> [!NOTE]
> **Use Network Activity:** If diagnosing connectivity or API issues, consult the Network Activity page for detailed ingress data.
