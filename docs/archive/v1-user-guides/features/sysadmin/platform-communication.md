# Platform Communication and Support

## Overview
As a System Administrator, you use these features to communicate wide-reaching changes via Global Announcements and address platform-level bugs submitted via User Reports.

## Step-by-Step Instructions

### Broadcasting a Global Announcement
1. Navigate to **Global Announcements** in the sysadmin sidebar.
2. Click **Create Announcement**.
3. Write a clear, friendly title and message.
4. Select the target audience (Teachers, Students, or Everyone). Ensure the message is relevant to the selected groups.
5. Set the **Expiration Date**. The announcement will automatically disappear after this date.
6. Click save to broadcast the announcement to the targeted dashboards.

### Managing Existing Announcements
1. Navigate to **Global Announcements**.
2. To pause an announcement without deleting it, click **Toggle**.
3. To update the text or extend the expiration date, click **Edit**.

### Reviewing and Resolving User Reports
1. Navigate to **User Reports** to view the queue of incoming tickets.
2. Select an issue to view the details, including the user's browser, OS, and their description of the problem.
3. To indicate you are investigating, change the status to **In Progress**.
4. Once the bug is fixed or the feedback is addressed, change the status to **Resolved**.

### Rewarding Good Bug Reports
To encourage helpful feedback, you can issue an economy reward to a user directly from their report.
1. Open a Resolved issue.
2. Select **Send Reward**.
3. Choose the token amount and note that it is for a helpful bug report. This automatically deposits funds into their checking account.

## Important Notes
> [!TIP]
> **Be transparent:** Use announcements to warn users *before* scheduled maintenance.

> [!NOTE]
> **Acknowledge feedback:** Try to resolve user reports quickly. Even if a feature request isn't implemented immediately, marking it "Resolved - Logged for Future" shows users their feedback was heard.
