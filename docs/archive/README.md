# Archive Documentation

This directory contains historical documentation, bug fix reports, and PR summaries from the project's development history.

## Structure

### [pr-reports/](pr-reports/)
Historical pull request descriptions, code reviews, and deployment instructions from major features and fixes.

### [releases/](releases/)
Detailed release notes for all versions, providing context and migration guides for each release.

### Root Level
Individual bug fix reports, migration summaries, and implementation reports from significant project milestones.

## Purpose

Archive documentation:
- Preserves context for historical decisions
- Documents resolved issues and their fixes
- Provides migration path references
- Maintains project history and rationale

## Usage

Reference archive documentation when:
- Understanding why a decision was made
- Investigating similar bugs or issues
- Learning about the system's evolution
- Researching migration patterns

For current documentation, see the main [Documentation Index](../README.md).

---

**Note:** Archive documents are maintained for historical reference and are not updated retroactively.
