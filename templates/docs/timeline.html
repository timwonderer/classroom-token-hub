Genesis
The Original Problem & Design Decisions
Pre-2024 → November 2025
Philosophy Architecture
expand_more

The project began as a full-time high school teacher's answer to a gap in educational technology: no platform simulated classroom economies realistically while respecting student data privacy.

"I will just make it myself."

Five Core Design Decisions Crystallized:
  • Experience Before Explanation — Students understand systems after feeling them, not from lectures
  • Join Code as Source of Truth — Early designs tied students to teacher accounts and failed in multi-teacher classrooms; join code became the absolute source of class isolation, reshaping the entire data model
  • Teacher Authority with Transparency — No invisible overrides; all teacher actions visible to students
  • Classroom Wage Index (CWI) — Macro-level stabilizer to prevent runaway wages and economic collapse
  • Insurance as Reality Response — Attendance isn't purely a moral choice; punishment disguised as realism is neither accurate nor educational
The Name Change: "Classroom Economy" → "Classroom Token Hub"

Not just a rebrand — a philosophical shift from a gimmick to a persistent, interconnected system. A hub for comprehensive financial literacy.

Key Learning: Real students find real bugs that simulated testing misses. Security must be foundational, not retrofitted.

crisis_alert Era II — Stability & Crisis Resolution
P0 Crisis
Same-Teacher Multi-Period Data Leak
November 2025
Critical Fix Security
expand_more
Impact: Students in multiple class periods with the same teacher saw aggregated financial data across all periods — a fundamental privacy and fairness violation.

Root Cause:
  • Transaction model had teacher_id but not join_code
  • All queries filtered by teacher_id only, returning data across all periods
Resolution:
  • Added join_code to all financial tables
  • Updated session management to track current_join_code
  • Modified ALL queries to scope by join_code
  • Created interactive backfill for legacy transactions
  • Added comprehensive multi-tenancy regression tests
Lesson: Multi-tenancy must be enforced at the database level, not the application level. join_code is non-negotiable.

Crisis
The Great Decimal Precision Crisis
January 2026
Critical Fix
expand_more
Symptoms: Zero-balance transfers triggering $35 overdraft fees • Partial rent payments leaving unpayable tiny balances ($0.0000001) • Dashboard interest calculations crashing with TypeErrors

Root Cause:
  • Transaction.amount stored as Float instead of Numeric
  • IEEE 754 floating-point: 0.1 + 0.2 ≠ 0.3
  • Mixed Decimal/float arithmetic caused TypeErrors
Lesson: Financial systems MUST use Decimal type, never Float. Precision errors compound invisibly and become hidden bugs.

lightbulb Design Philosophy — What We Refuse to Build

Ten deliberate absences that define Classroom Token Hub as much as what it includes

❌ NO XP BARS OR STREAKS Earned wages and time-based pay — not gamification ✅ Instead: Real wages, real consequences
❌ NO PERFECT ATTENDANCE BONUSES Life interferes; punishment isn't realism ✅ Instead: Insurance system handles legitimate absences
❌ NO INVISIBLE TEACHER OVERRIDES No silent balance edits; no hidden forgiveness ✅ Instead: All actions logged and visible to students
❌ NO BEHAVIOR SCORES No engagement ratings or compliance metrics ✅ Instead: Neutral logs supporting teacher conversations
❌ NO AI JUDGMENTS OF INTENT No motivation predictions or effort ratings ✅ Instead: Neutral logs that don't infer character
❌ NO UNLIMITED CONFIG KNOBS Arbitrary overrides break system coherence ✅ Instead: Guardrailed settings that preserve integrity
❌ NO FRICTIONLESS SHORTCUTS One-click forgiveness undermines the system ✅ Instead: Deliberate processes requiring acknowledgment
❌ NO LEADERBOARDS Public rankings don't belong in classrooms ✅ Instead: Private balances protect student dignity
❌ NO NEUTRALITY CLAIM All systems have assumptions ✅ Instead: Visible assumptions open to class discussion
❌ NO DIRECT STUDENT → SYSADMIN TICKETS Teachers are the first line of support ✅ Instead: Teacher-mediated escalation preserves authority
"Classroom Token Hub was created to fill the void of a platform that simulates an economy more realistically than what is available on the market. It also functions as a classroom management tool that teaches students personal responsibility. Most of all, it respects data privacy and security like no other. The friction you experience when creating, claiming, and resetting accounts was intentional. Classroom Token Hub does not ask, 'How can we defend ourselves?' It was designed to anticipate, 'What if the bad actor gets in?'"
— From Project History & Philosophy
