How to: Student Management

Use this hub to add, update, move, and remove students across class periods. The roster details you enter here are the exact details students must use to claim their accounts.

Choose the method that matches your situation:

  • Bulk Upload: Best for full rosters. Download the template from the Bulk Upload tab, fill it out, and upload it back.
  • Individual Upload: Best for new students who join mid-year or one-off updates.
Roster accuracy checklist:
  • Do not modify the header row
  • Date of Birth must be in MM/DD/YYYY format (matches student claim entry)
  • Students must be assigned to a class section

When adding a single student, use the Individual Upload form and select the correct class section. If the class does not exist, create the section first.

Quick Start: Students claim their account by clicking I need a student account on the student login page.

What students need:

  • Join Code — Found in the Class Roster tab for each class section
  • First Name — As entered in your roster
  • Last Name — As entered in your roster
  • Date of Birth — The exact date you entered

Account Setup Steps:

  1. Student enters their information and gets a theme word prompt
  2. A unique username is auto-generated using their theme word
  3. Student sets a 4-8 digit PIN (for quick access)
  4. Student creates a long passphrase (for security)
Critical: The username is shown only once during setup. Students must record it. If forgotten, you'll need to reset their account.
Teacher Tip: Have students complete account setup in class so you can help if they have issues. Although not recommended for security reasons, consider keeping a secure record of usernames for your class so you can assist with forgotten usernames.

What you can update: First name, last name, date of birth, and class assignment.

How to update:

  1. Go to the Class Roster tab
  2. Click Edit next to the student's name
  3. Update their information
  4. Click Save. The changes take effect immediately
Changing Class Assignment: If you move a student to a different class period, you can choose to transfer their balance or start fresh at $0.
Important: If you change their name or date of birth, the student will need these updated details to log in. Unclaimed students will use the new info during account setup.

Use reset when a student forgets credentials or cannot log in. Resetting preserves account data but forces a new claim.

Resetting requires a new claim: The student must re-claim using the join code and their updated details.

After the reset, have the student follow the normal claim process on the student login page.

To remove a student, use the Remove action next to their name in the Class Roster.

Important: Removing a student is permanent and deletes their access to the class.

If you only need to move them, use Edit to change their class section instead.

Use the Export All button in the Class Roster header to download a CSV of your students.

Tip: Exports are helpful for reviewing roster data, keeping offline backups, or preparing a clean bulk upload.
